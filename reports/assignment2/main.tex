\section{Dataset \& Setup}
This project focuses on the AGNews Dataset, created by Zhang et al. \cite{AGNews}. The dataset consists of 120k training datapoints and 7.6k testing points. Each datapoint consists of a title and a description of a news article. These are around 45 and 200 characters long on average. Each article is categorised based on topic. The set of labels consists of World, Sports, Business and Sci/Tech ($C=4$, $y\in\{1,2,3,4\}$). This project aims to investigate the performance of baseline models on this task. For hyperparameter tuning and testing, a random development split of 10\% was taken from the training set.

To train models, first, the data needs to be preprocessed. The first step was to concatenate the title and the description into a single string of text, which makes it easier to use both features. After this, some basic preprocessing is done, removing short words 

Next, an embedding needs to be made. For this purpose, a Word2Vec vectorizer was trained on the training data, after filtering out words that appear less than 3 times. An embedding dimension of 100 was used; the vectorizer was trained using a window of length 5 across 10 epochs. 

Every process is carried out with a set random seed; 33 was chosen for this.

\section{Models}
This report details an ablation study on sequence length. This is done for two types of models: a recurrent Long Short-Term Memory (LSTM) Model and a Convolutional Neural Network. Both models are used to encode the sequence, after which a linear model extracts the prediction. Detailed in Equation \eqref{eq:linmodel}. The exact dimensions of $\boldsymbol{W}$ differ for the LSTM and CNN models.

\begin{equation}
    \boldsymbol{a} = \boldsymbol{W}\boldsymbol{x}
    \label{eq:linmodel}
\end{equation}

This results in a vector of 4 logits. To convert these logits to a probability, the softmax activation function is used, specified in Equation \eqref{eq:softmax} \cite{pml1Book}.

\begin{equation}
    p(y=c|\boldsymbol{x}, \boldsymbol{W}) = \frac{e^{a_c}}{\sum_{c'=1}^C e^{a_{c'}} }
    \label{eq:softmax}
\end{equation}

\subsection{CNN}
The architecture used for the CNN focuses on a 1D convolution operation across the sequence. This is done for five filters of different lengths, from 3-7 words long. The convolution operation results in a new sequence of vectors $\boldsymbol{h}(t)$ where $t$ specifies the index in the sequence \cite{Goodfellow-et-al-2016}.

\begin{equation}
    \boldsymbol{h}(t) = (\boldsymbol{x} * \boldsymbol{w})(t) = \sum_n x(t+n)w(t)
\end{equation}

Thus, after applying each of the five convolutions, we are left with five sequences, which are pooled together using a max-pooling operation to five vectors of dimension 100; the ReLU activation function (Equation \eqref{eq:relu}) is called on each. Lastly, these vectors are concatenated and form the input for the final linear model in Equation \eqref{eq:linmodel}.

\begin{equation}
    \text{ReLU} (x) = \max(0,x)
    \label{eq:relu}
\end{equation}

The weights of the CNN were initialised using Xavier Uniform initialisation. Here, weights are sampled from a uniform distribution with bounds based on the model complexity. The exact architecture is presented in detail in Table \ref{tab:cnn-architecture} of Appendix \ref{ap:arch}.

\subsection{LSTM}
The LSTM is a gated Recurrent Neural Network (RNN). The LSTM, aside from the standard hidden state vector $\boldsymbol{h}_t$, also has a cell state $\boldsymbol{c}_t$. Like in many RNNs the input $\boldsymbol{x}_t$ is combined with the last cell state $\boldsymbol{h}_{t-1}$ to provide the input for that timestep $t$. An LSTM sets itself apart by its usage of gates, which determine what information from the cell state is forgotten, what information is added and what information is used to construct the hidden state. These gates are labeled $\boldsymbol{f}_t$, $\boldsymbol{i}_t$ and $\boldsymbol{o}_t$, respectively. This results in a set of update equations shown below \cite{jm3}:

\begin{align}
    \boldsymbol{f}_t &= \sigma(\boldsymbol{U}_f\boldsymbol{h}_{t-1} + \boldsymbol{W}_f\boldsymbol{x}_t) \\
    \boldsymbol{i}_t &= \sigma(\boldsymbol{U}_i\boldsymbol{h}_{t-1} + \boldsymbol{W}_i\boldsymbol{x}_t) \\
    \boldsymbol{o}_t &= \sigma(\boldsymbol{U}_o\boldsymbol{h}_{t-1} + \boldsymbol{W}_o\boldsymbol{x}_t) \\
    \boldsymbol{g}_t &= \tanh(\boldsymbol{U}_g\boldsymbol{h}_{t-1} + \boldsymbol{W}_g\boldsymbol{x}_t) \\
    \boldsymbol{c}_t &= \boldsymbol{c}_{t-1} \odot \boldsymbol{f}_t + \boldsymbol{g}_t\odot \boldsymbol{i}_t \\
    \boldsymbol{h}_t &= \boldsymbol{o}_t \odot \tanh(\boldsymbol{c}_t)
\end{align}

Where $\boldsymbol{U}$ and $\boldsymbol{W}$ are matrices and $\odot$ signifies the elementwise (Hadamard) product. The sigmoid function $\sigma(x)$ is given by Equation \eqref{eq:sigmoid}.

\begin{equation}
    \sigma(x) = \dfrac{e^x}{1+e^x}
    \label{eq:sigmoid}
\end{equation}

In this report, a bidirectional version of this system is used, which entails that the input signal is fed through the system both forwards and backwards, and the hidden states are concatenated. Additionally, a layered structure is used. This means that four LSTM-Cells as detailed above were used. The hidden cells $h_{t}^i$ of the $i$-th layer served as the input ($x_t$ in the equations above) for the  $i+1$'th layer. This structure is capable of capturing complex patterns more easily. The model used in the ablation study consisted of four hidden layers. Due to the increased complexity of the LSTM cells, this was chosen to provide a similar model complexity as the CNN. A more detailed tabular overview of the model architecture is presented in Table \ref{tab:lstm-architecture} of Appendix \ref{ap:arch}

\section{Training Protocol}
The models discussed above were implemented using PyTorch \cite{paszke2019pytorchimperativestylehighperformance}. As briefly mentioned before, the purpose of this report is to investigate the performance of the LSTM and CNN models for a varying sequence length. To do this, a model will be trained on data for each maximum sequence length $L \in \{32,64,128,256\}$ as part of the ablation study; with regard to results reported in the following section, we train with a maximum sequence length $L=256$. The sequences will be padded or cut down to the required length for ease of computation. Both models are trained using the same Word2Vec embedding. The model architectures discussed in Appendix \ref{ap:arch}, will remain constant. Additionally, for both models and all values $L$, the Adam optimiser is used with a starting learning rate of $10^{-3}$ and the cross-entropy loss function is applied. Lastly, training is done for a maximum of 20 epochs, with early stopping configured to stop training if the validation loss does not improve for 3 consecutive epochs. The model with the best validation loss is restored at the end of training.

\subsection{CNN}
Looking at the confusion matrix for the Test set of the CNN, it becomes clear that the most commonly made mistake is that the Business and Sci/Tech classes are confused with each other. This classification error also occurred for the baseline models in Assignment 1. The baseline models also showed a significant amount of classification errors between the categories World and Sports, the more complex CNN, however, shows a lower amount of errors between these categories and instead shows a higher error rate between the Business and World classes. The fact that the World - Sports classification error occurs less when it comes to the CNN could be explained by the fact that the context around a word matters more for the CNN. In assignment 1, it was made clear that specific sport-related terms could lead to a classification in the Sports class, while that may not be correct, including context for these examples leads to a lower error rate. \ref{tab:misclassificationCNN} mentions the 10 lowest confidence errors, taking a look at each of them shows us the following. 

In references 1 and 2, the actual class is Sports, but the predicted class is Sci/Tech. Since the CNN uses filters from 3 - 7 words long, we need to look at combinations of words that can cause this misclassification. For reference 1, this could be "successful US debut by beating Literacy". For reference 2, it could be "standing virtually in the middle". Reference 3 mentions "Just begin Glasgow, Cambridge and Cardiff", which is a way of wording that might occur in sports articles, causing a wrong Sports classification. Reference 4 mentions "Cruise line" and "service problems", which could be confused as Sci/Tech sentences rather than business-related topics. For references 5 and 9, there is mostly a mention of a specific person or a party/event. These examples do not really contain any specific sports terms; they are being wrongly classified as the World class. Reference 6 uses the words "buzz" and "cacophony", which may be the reason for a misclassification as Sci/Tech instead of Business. Sentences like "Planning to proceed with" and "gets official go-ahead of the votes" in reference 7 and 8 very likely appear more in business-related articles than sports-related articles, causing this classification error. Reference 10 is about a water crisis and does not have any mention of Sci/Tech-related terms, so a misclassification as World is not surprising. 


\subsection{LSTM}
For the LSTM a similar effect can be seen. Sci/Tech and Business are the most often confused. In comparison to the baseline models, the amount of misclassifications between Sports and World has also decreased for the LSTM. The second most made classification error is also between the classes Business and World. Just like for the CNN it can be stated that an LSTM relies on context and previous inputs. This means that individual words matter less in the classification and instead word combinations and specific ways of wording sentences matters more. We take a look at \ref{tab:misclassificationLSTM} for 10 specific cases of misclassification and why they could be occurring. 

References 1 and 5 misclassify a Business article as a Sports article. For reference 1, this could be caused by the sentence "downgrading of San Diego's credit rating" while for reference 5, the cause may be a sentence like "The green is still there" or "Stars and Stripes now waves in the background". For reference 2, it is quite difficult to see why the misclassification occurs. The article talks about a lawsuit regarding a governor violating the separation of powers, which does not necessarily sound like a business-related article. We were unable to find a plausible explanation for this missclassification. 

References 3 and 4 have the predicted class Sci/Tech, while the actual class is World. Reference 3 features sentences such as "a guide to the primary get ready", while reference 4 mentions a "Potemkin village Gongzhong". Reference 6 is about a "plan to build a multi-million-dollar race track" and a bid that has advanced far; these are sentences likely to occur in Business articles and less likely in Sport articles, thus causing misclassification. References 7 and 10 misclassify a Sci/Tech article as a Business article, thus we are once again looking for sentences that would regularly occur in business-related articles. For reference 7, this could be "Fabled Hollywood producer and man about town".

For reference 10 it is likely "TravelWatch column" or "Aborigine-owned tour company". Reference 8 does not necessarily contain any sentences that would cause a classification as Sci/Tech. There are no mentions of any technological or scientific phenomena and instead only mentions of lacking funds and world population growth, which is more likely to fall into the World class. Lastly, reference 9 does not contain any Business related words or sentences and instead feautures mentions of "The U.S. Coast Guard" and "States of Alabama, Florida and Mississipi", this causes the LSTM to misclassify the article under the World class.

\subsection{Comparison to Assignment 1}
With respect to Assignment 1, it becomes clear that the misclassifications occur less on singular words, but rather on the context that the sentences appear in. This is logical considering that both the CNN and the LSTM rely on the context of the sentence. The CNN does this using a window over multiple words, while the LSTM remembers or forgets previous words and uses them as context. It can also be seen that the accuracies are higher for the more complex models than the baseline models, as is to be expected. 

\section{Ablation Study Concerning Max Sequence Length}

An ablation study was run concerning the max sequence length with the following values: 16, 32, 64, 128, and 256 tokens. With the configuration, we followed the training protocol as standard, with only the max sequence length being changed with the dataset. The initial hypothesis here is that longer sequences provide the model with more context and we expect a higher accuracy. Table \ref{tab:seq_length_comparison} displays the accuracy results per model per dataset per max sequence length, and Figures \ref{fig:cnn_maxlen} and \ref{fig:lstm_maxlen} display the training history per model across all tested max sequence lengths. 

\begin{table}[h]
    \centering
    \begin{tabular}{|c|c|c|c|}
        \hline
        \textbf{Max Sequence Length} & \textbf{Model} & \textbf{Accuracy (Dev Set)} & \textbf{Accuracy (Test Set)} \\ \hline
        \multirow{2}{*}{16}  & CNN  & 0.8721 & 0.8977 \\ \cline{2-4} 
                             & LSTM & 0.9038 & 0.9300 \\ \hline
        \multirow{2}{*}{32}  & CNN  & 0.8858 & 0.9133 \\ \cline{2-4} 
                             & LSTM & 0.9011 & 0.9255 \\ \hline
        \multirow{2}{*}{64}  & CNN  & 0.8888 & 0.9113 \\ \cline{2-4} 
                             & LSTM & 0.9016 & 0.9272 \\ \hline
        \multirow{2}{*}{128} & CNN  & 0.8837 & 0.9101 \\ \cline{2-4} 
                             & LSTM & 0.9013 & 0.9286 \\ \hline
        \multirow{2}{*}{256} & CNN  & 0.8861 & 0.9091 \\ \cline{2-4} 
                             & LSTM & 0.9003 & 0.9313 \\ \hline
    \end{tabular}
    \caption{Model Performance Comparison across Maximum Sequence Lengths}
    \label{tab:seq_length_comparison}
\end{table}

For the CNN model, the highest accuracy for the dev set was at a maximum sequence length of 64, at 0.8888. The highest accuracy for the test set was at a maximum sequence length of 32, at 0.9133. So, the CNN model appears to perform best on the test set with shorter sequences, but needs more context to reach its peak on the dev set. For the LSTM model, the highest accuracy for the dev set was at a maximum sequence length of 16, at 0.9038. The highest accuracy for the test set was at a maximum sequence length of 256, at 0.9313. So, the LSTM model appears to perform best at a maximum length.

The results of this ablation study present a counterintuitive trend. Typically, an ablation study on sequence length is expected to highlight a significant degradation in model performance as the available context is reduced. However, our results demonstrate a surprising resilience and powerful performance at lower maximum sequence lengths. We identified the reasoning for this: In our preprocessing, we concatenate the title and description, which results in the most contextual information being processed regardless of sequence length. So, by our own preprocessing decisions, the effect of the ablation did not result in a significant decline in the performance of the two models. 

\section{Discussion}
This report aimed to investigate the CNN and LSTM model architectures' performance on the AG News dataset. Both models performed well and beat the baseline model's performance, as demonstrated by assignment one's results in Figure \ref{fig:assignment_1}. When examining the errors, most mistakes from the CNN can be traced back to specific words in different filter lengths of the sentence. It is expected that this can be helped by feeding the convolved signals into each other rather than pooling over all the filters. This way, the present context of different parts of the sentence can be combined more easily. This can be seen as the LSTM, which does contain multiple layers which transform the input to achieve better accuracy than the CNN. Despite these differences in accuracy the models do not differ wildly in the types of errors they make.

As for the ablation study, the results did not match the initial hypothesis. The data shows a strong resilience of the model to the shorter sequence lengths. We expect this is in part due to the way we structured the data preprocessing. As the data was preprocessed by concatenating the title and the description, while the sequences were cut by taking the first n tokens from the start, the shorter sequences ended up storing similar information. We expect that when changing the preprocessing to only include the description or when taking the sequence from the centre and not the start, that the hypothesised decrease in performance will be observed.
